1Microsoft 365 Businessアカウントの作成

🎯 この手順の目的

TeamsのボットをテストするためのMicrosoft 365テナント（組織環境）を用意します。個人のTeamsアカウントではボット開発に必要な機能が制限されるため、Business/Enterpriseプランが必要です。

📋 アカウント作成手順

  1. ブラウザで microsoft.com/ja-jp/microsoft-365/business にアクセス
  2. 「Microsoft 365 Business Standard」または「Business Basic」の「無料試用版」をクリック
  3. メールアドレスを入力（GmailなどでもOK）
  4. 「職場または学校のアカウントの設定」に進む
  5. 会社名・ドメイン名を入力（例：yourname.onmicrosoft.com）
  6. 管理者ユーザー名とパスワードを設定してアカウント作成
💡 無料試用版は30〜90日間使用可能です。開発・テスト目的であれば十分な期間です。

✅ 確認ポイント

作成後、admin.microsoft.com（Microsoft 365 管理センター）にログインできればOKです。

2Teamsの動作確認

🎯 この手順の目的

作成したテナントでTeamsが使えることを確認します。後でボットをこのTeams環境に追加します。

📋 手順

  1. ブラウザで teams.cloud.microsoft にアクセス
  2. 作成した管理者アカウントでサインイン（例：admin@yourname.onmicrosoft.com）
  3. Teamsのチャット画面が表示されればOK
💡 Teamsデスクトップアプリがある場合は、そこからサインインしてもOKです。

3Azureサブスクリプションの登録

🎯 この手順の目的

Azure OpenAIやボットのバックエンドをホストするためのAzureアカウントを用意します。M365とAzureは別サービスのため、別途登録が必要です。

📋 無料アカウントでの登録手順

  1. ブラウザで portal.azure.com にアクセス
  2. 「無料で始める」をクリック
  3. Microsoftアカウントでサインイン（M365のアカウントでもOK）
  4. 氏名・電話番号・クレジットカード情報を入力（本人確認用、すぐ課金はされない）
  5. 利用規約に同意して登録完了
💡 無料アカウントでは $200クレジット（30日間） と、12ヶ月間の主要サービス無料枠が付与されます。ハッカソン開発程度であれば無料枠内で十分です。
⚠️ 過去にAzure無料アカウントを使用したことがある場合は「対象外」と表示されることがあります。その場合は「従量課金（Pay-as-you-go）」を選択してください。ハッカソン期間中の実費は数百〜1,000円程度です。

✅ 確認ポイント

Azureポータルのホーム画面（クイックスタートセンター）が表示されればOKです。

4Azure AI Foundryリソースの作成

🎯 この手順の目的

Azure AI Foundryは、OpenAIモデルを含むAIサービスを一元管理するプラットフォームです。GPT-4oなどのモデルをデプロイして、アプリから呼び出せるようにします。

📋 リソース作成手順

  1. Azureポータル上部の検索バーに「Azure OpenAI」と入力
  2. 「Microsoft Foundry | Azure OpenAI」ページを開く
  3. 「+ 作成」→「Foundry（おすすめ）」をクリック
  4. 以下の項目を入力：
サブスクリプション:     （自分のサブスクリプションを選択）
リソースグループ:       「新規作成」→ 任意の名前（例: hackathon-rg）
名前:                   任意の名前（例: hackathon-foundry）※グローバルで一意な名前
リージョン:             Japan East（日本から使う場合はここが最速）
Default project name:   任意（例: proj-default）
  • 5. 「確認と作成」→「作成」をクリック
  • 6. デプロイ完了まで1〜3分待つ
    • 💡 リソース名はURLに使われるため、英数字とハイフンのみ使用可能です。「hackathon-202604」のような形式がおすすめです。

    ✅ 確認ポイント

    概要ページで「状態：Succeeded」と表示されればリソース作成完了です。

    5GPT-4oモデルのデプロイ

    🎯 この手順の目的

    リソースを作っただけではモデルは使えません。使いたいモデル（GPT-4o）を「デプロイ」することで、APIから呼び出せるようになります。

    📋 デプロイ手順

    1. 作成したFoundryリソースの概要ページで「Foundry ポータルに移動」をクリック
    2. 左サイドバーの「モデル カタログ」を開く
    3. 検索欄に「gpt-4o」と入力して検索
    4. 「gpt-4o」のカードをクリック
    5. 「このモデルを使用する」をクリック
    6. デプロイ設定を確認して「デプロイ」をクリック
    デプロイ名:   gpt-4o（デフォルトのままでOK）
モデル版:     2024-05-13（最新の安定版）
展開の種類:   Standard
    💡 デプロイ名は後でコード内で使います。わかりやすい名前（gpt-4o）のままにしておくと管理が楽です。

    ✅ 確認ポイント

    デプロイ一覧で「プロビジョニングの状態：成功」と表示されればOKです。

    6APIキーとエンドポイントの取得

    🎯 この手順の目的

    コードからAzure OpenAIを呼び出すために必要な「エンドポイントURL」と「APIキー」を取得します。

    📋 取得手順

    1. Azureポータルで作成したFoundryリソースを開く
    2. 左サイドバー「リソース管理」→「キーとエンドポイント」をクリック
    3. 「キーの表示」ボタンをクリック
    4. 「OpenAI」タブをクリック
    5. 以下の2つをメモ・コピーしておく：
    ① キー1（または キー2）   ← 秘密にすること！GitHubに上げない！
② 言語 API エンドポイント  ← https://[リソース名].openai.azure.com/
    ⚠️ APIキーは絶対に公開しないでください。GitHubにアップする場合は .gitignore でキーを含むファイルを除外する必要があります。

    🎯 この手順の目的

    Teamsアプリ開発に必要なツールをVS Codeに追加します。

    4. Microsoftが公開している拡張機能を「インストール」
    💡 旧名は「Teams Toolkit」です。検索しても出てこない場合は「Teams Toolkit」でも探してみてください。

    8Teamsエージェントプロジェクトの生成

    🎯 この手順の目的

    Toolkit付属のテンプレートから、Teams上で動くAIエージェントの雛形コードを自動生成します。

    📋 プロジェクト生成手順

    1. VS Code左サイドバーのM365アイコン（四角いロゴ）をクリック
    2. 「新しいエージェント/アプリの作成」をクリック
    3. テンプレートを順番に選択：
    ① カテゴリ:     Teams エージェントとアプリ
② 種類:         一般的なTeamsエージェント
③ LLMサービス: Azure OpenAI
④ APIキー:      Enterでスキップ（後で設定）
⑤ 言語:         Python
⑥ 保存先:       任意のフォルダ

    プロジェクトフォルダが自動で生成され、VS Codeで開かれます。

    💡 生成されるファイル構成：
    appPackage/ — Teamsアプリのマニフェスト
    env/ — 環境変数ファイル群
    src/ — ボットの本体コード
    infra/ — Azureデプロイ用インフラ定義

    9環境変数の設定と依存ライブラリのインストール

    🎯 この手順の目的

    STEP 6で取得したAPIキーとエンドポイントをプロジェクトに設定し、必要なPythonライブラリをインストールします。

    📋 環境変数の設定

    プロジェクト内の env/.env.local.user をVS Codeで開き、以下を編集します：

    SECRET_AZURE_OPENAI_API_KEY=（STEP6でコピーしたキーを貼り付け）
AZURE_OPENAI_ENDPOINT='https://[リソース名].openai.azure.com/'
AZURE_OPENAI_DEPLOYMENT_NAME='gpt-4o'
    ⚠️ このファイルは .gitignore で除外されているため、Gitにコミットされません。チームで共有する場合はセキュアな方法（Azure Key Vault等）を使ってください。

    📋 依存ライブラリのインストール

    VS Codeのターミナルを開き（Ctrl+`）、プロジェクトフォルダで以下を実行します：

    python -m pip install -r src/requirements.txt

    エラーが出なければインストール完了です。

    🚀 次のステップ：ローカルでボットを起動する

    環境変数の設定とライブラリのインストールが完了したら、F5キーを押してデバッグを開始します。Toolkitが自動でTeams環境と接続し、ブラウザ上のTeamsでボットが起動します。

    💡 初回起動時はBot FrameworkやDev Tunnelの認証が求められる場合があります。画面の指示に従ってサインインしてください。

    10ローカルデバッグの起動とPlaygroundでの動作確認

    🎯 この手順の目的

    F5キーでボットをローカル起動し、Microsoft 365 Agents Playgroundを使ってTeams上の動作をシミュレートします。Azureへのデプロイ前にコードを素早く検証できます。

    ⚠️ よくあるエラー：ModuleNotFoundError

    F5起動時に ModuleNotFoundError: No module named 'azure' が表示される場合、デバッガーが使用するPythonインタープリターと、ライブラリをインストールしたPython環境が異なることが原因です。

    ターミナルのエラーメッセージに表示されているPythonのフルパスを使って、再度インストールします：

    # エラーメッセージ内のPythonパスを使ってインストール（例）
& 'C:\Users\[ユーザー名]\AppData\Local\Programs\Python\Python313\python.exe' -m pip install -r src/requirements.txt
    💡 VS Codeで使用するPythonインタープリターは Ctrl+Shift+P → 「Python: インタープリターを選択」で確認・変更できます。

    ⚠️ よくあるエラー：ポート3978が使用中

    F5を複数回押すと OSError: [WinError 10048]（ポート3978が既に使用中）というエラーが発生します。

    デバッグツールバーの ■（停止）ボタン を押してすべてのデバッグセッションを終了してから、再度F5を押してください。

    📋 ローカルデバッグの正しい起動手順

    1. VS Codeで プロジェクトフォルダ（Hackathon2026）を直接開く（親フォルダではなく、プロジェクト直下を開くこと）
    2. エディタ上のタブをすべて閉じ、ファイルを選択していない状態にする
    3. F5キーを押す（または「実行」→「デバッグの開始」）
    4. ブラウザが自動で開き、Microsoft 365 Agents Playground が表示される
    5. 「Personal Chat」をクリックし、メッセージを送信して返答が返ってくることを確認
    💡 Playgroundは本物のTeamsを使わずにボットの動作をシミュレートできるブラウザ上のテスト環境です。Azure接続なしで動作するため、開発サイクルが速くなります。

    ✅ 確認ポイント

    Playgroundの右側「Log Panel」に Connected. と表示され、チャットでメッセージを送ると返答が返ってくればローカル動作確認完了です。

    11Teams管理センターでカスタムアプリのアップロードを有効化

    🎯 この手順の目的

    Toolkitを使って本物のTeamsにボットをデプロイするには、テナントでカスタムアプリのアップロードが許可されている必要があります。デフォルトでは無効のため、Teams管理センターで設定を変更します。

    📋 設定手順

    1. ブラウザで admin.teams.microsoft.com にアクセス
    2. 管理者アカウントでサインイン
    3. 左メニューの「Teams のアプリ」→「セットアップ ポリシー」をクリック
    4. 「グローバル（組織全体の既定値）」をクリックして開く
    5. 「カスタムアプリのアップロード」トグルを オン に切り替える
    6. 「保存」をクリック
    ⚠️ この設定は反映まで 最大24時間 かかる場合があります。設定後すぐに反映されない場合は時間をおいて再試行してください。
    💡 VS Code側でエラーが出る場合は、左サイドバーのM365アイコン → ACCOUNTSでアカウントを一度サインアウト→再サインインすると解消されることがあります。

    ✅ 確認ポイント

    VS CodeのM365 Agents ToolkitパネルのACCOUNTSセクションで「Custom App Upload Enabled」と表示されれば設定完了です。

    12AzureへのProvision（準備）とDeploy（配置）

    🎯 この手順の目的

    ローカルで動作確認できたボットを、本物のTeams環境で使えるようにAzureにデプロイします。ToolkitのLIFECYCLE機能を使うと、Azure Bot Serviceの登録やApp Serviceへのデプロイが自動で行われます。

    📋 Provision（準備）手順

    Azureにリソースを作成し、ボットを登録します。

    1. VS CodeでプロジェクトフォルダをワークスペースとしてM365 Agents Toolkitを開く
    2. 左サイドバーのM365アイコンをクリック
    3. 「LIFECYCLE」セクションの「準備」をクリック
    4. サインインを求められた場合はM365管理者アカウントでサインイン
    5. Azureサブスクリプションとリソースグループを選択
    6. プロビジョニング完了まで待つ
    💡 Provisionでは Azure Bot Service・App Service などのリソースが自動作成されます。完了後に env/.env.dev にBOT_IDなどの値が自動で書き込まれます。

    📋 Deploy（配置）手順

    ローカルのコードをAzureのApp Serviceにアップロードします。

    1. 「LIFECYCLE」セクションの「配置」をクリック
    2. デプロイ完了のメッセージが表示されるまで待つ
    配置ステージ 2/2 アクションが正常に実行されました。

    上記メッセージが表示されればデプロイ成功です。

    📋 Teamsへのインストール手順

    1. 「LIFECYCLE」セクションの「組織に公開」をクリック
    2. または Teamsを開き「アプリ」→「カスタムアプリをアップロード」でzipファイルを選択
    3. zipファイルは appPackage/build/ フォルダ内に生成されています
    💡 初回デプロイ後、Teamsにボットが表示されるまで数分かかる場合があります。

    ✅ 次のステップ

    本物のTeamsでボットとチャットできることを確認したら、次は以下の機能追加に進みます：

    • Azure AI Speech を使った音声入出力（STT / TTS）の追加
    • Teams Tabアプリとしてマイク入力UIの実装
    • ACS（Azure Communication Services）を使った会議参加・音声文字起こし

    13Dev Tunnelを使った本物のTeamsでの動作確認

    🎯 この手順の目的

    AzureへのProvisionがApp Serviceのクォータ制限で失敗する場合でも、Dev Tunnelという機能を使えばローカルで動作するボットを本物のTeamsに接続できます。ボットはPCで起動し続けている必要がありますが、開発・テスト用途には十分です。

    ⚠️ よくあるエラー：.env.dev.user の環境変数が空

    Provisionを実行すると arm.MissingEnvironmentVariablesError が出る場合があります。Azureデプロイ用の環境変数ファイル（env/.env.dev.user）が空のためです。

    env/.env.local.user と同じ内容を env/.env.dev.user にも設定してください：

    SECRET_AZURE_OPENAI_API_KEY=（Azure OpenAIのAPIキー）
AZURE_OPENAI_ENDPOINT='https://（your-resource）.openai.azure.com/'
AZURE_OPENAI_DEPLOYMENT_NAME='gpt-4o'
    💡 ローカル用（.env.local.user）と本番用（.env.dev.user）は別ファイルです。両方に設定が必要です。

    ⚠️ よくあるエラー：App Service クォータ不足

    新規サブスクリプションでは、App Service Plan（F1/B1/S1など全プラン）のVM クォータが 0 に設定されており、armDeploy.DeployArmError が発生します。Azure ポータルのクォータページ（App Service Public Preview）でも申請ボタンがグレーアウトされており、現時点での申請は不可能です。

    ⚠️ この問題はAzureサブスクリプション作成直後に発生しやすいです。時間をおいて再試行するか、下記のDev Tunnelを使った方法で回避してください。

    📋 Dev Tunnelを使ってTeamsで起動する手順

    1. VS Codeの上部デバッグ設定ドロップダウン（▼）をクリック
    2. 「Debug in Teams (Desktop)」 を選択（Playgroundではなく Teams）
    3. F5 を押す
    4. 自動でプロビジョニング（6/6ステージ）と配置（1/1ステージ）が実行される
    5. Teamsデスクトップアプリが起動し、ボットのインストールダイアログが表示される
    6. 「追加」をクリックしてボットをTeamsに追加
    7. Teamsの左サイドバー下部「アプリ」からボットを呼び出す
    💡 Dev Tunnelはローカルで動作するボットをインターネット経由でTeamsに公開するトンネル技術です。App Serviceのクォータ問題を回避でき、Azureのリソース料金も発生しません。

    ✅ 動作確認

    Teamsのチャット画面でボットにメッセージを送り、GPT-4oによる返答が返ってくることを確認します。

    「AI により生成」と表示され、GPT-4oの返答が届いていれば成功です。

    📝 この方法の制限事項

    • VS Codeとローカルのボットサーバーが起動している間だけTeamsで使用できる
    • PCをシャットダウンするとボットが応答しなくなる
    • 永続的な公開にはAzure App Serviceへのデプロイが必要（クォータ問題解決後）
